# 주식 분석 AI 시스템 요건정의서

**작성일:** 2026-04-08  
**버전:** 1.0

---

## 1. 프로젝트 개요

네이버 금융 애널리스트 리포트, DART 공시, 재무 데이터, 뉴스를 종합적으로 수집·분석하여, AI 에이전트가 스스로 질문을 생성하고 웹 검색을 통해 정교한 주식 분석 보고서(Report + PPT)를 자동 작성하는 시스템을 구축한다.

## 2. 데이터 수집 요건

### 2.1 네이버 금융 애널리스트 리포트 수집 (요건 1)

| 항목 | 내용 |
|------|------|
| 대상 URL | https://finance.naver.com/research/company_list.naver |
| 수집 형식 | PDF |
| 수집 방법 | 웹 스크래핑 후 PDF 다운로드 |
| 수집 주기 | 일 1회 (신규 리포트 감지 시) |
| 저장 방식 | 로컬 파일시스템 + 벡터 DB (RAG용) |

### 2.2 장기 데이터 분석 및 분류 (요건 2)

| 항목 | 내용 |
|------|------|
| 대상 기간 | 1년 이상 |
| 분류 기준 | 섹터, 시가총액, 투자의견(Buy/Hold/Sell), 목표주가 상향/하향 |
| 분석 목적 | 트렌드 파악, 섹터별 애널리스트 컨센서스 변화 추적 |
| 출력 | 분류된 종목 리스트 및 요약 통계 |

### 2.3 관심 종목 뉴스 수집 (요건 3)

| 항목 | 내용 |
|------|------|
| 수집량 | 하루 10개 종목 × 주요 뉴스 |
| 수집 기준 | 관심종목 리스트 기반 필터링 |
| 뉴스 소스 | 네이버 뉴스, 주요 경제지 |
| 중요도 판단 | LLM이 헤드라인 및 본문 요약 후 관련성 스코어링 |
| 수집 주기 | 일 1회 (장 마감 후) |

### 2.4 DART 공시 데이터 수집 (요건 4)

| 항목 | 내용 |
|------|------|
| 수집 대상 | 사업보고서, 분기보고서, 주요사항보고서, 공정공시 |
| API | DART Open API (https://opendart.fss.or.kr) |
| **필요 이유** | 재무제표 원본 데이터(영업이익·부채비율 등) 확보, 네이버 금융에 반영되지 않은 최신 공시(유상증자·CB발행 등 주가 영향 이벤트) 실시간 감지, 애널리스트 리포트의 수치 검증 |
| 수집 주기 | 일 1회 + 실시간 주요공시 알림 |

### 2.5 네이버 주식 재무 분석 및 핵심 지표 (요건 5)

수집 대상 지표:

- **밸류에이션:** PER, PBR, PSR, EV/EBITDA
- **수익성:** 영업이익률, ROE, ROA, EBITDA 마진
- **성장성:** 매출 YoY, 영업이익 YoY, EPS 성장률
- **안정성:** 부채비율, 유동비율, 이자보상배율
- **시장 데이터:** 시가총액, 외국인 보유비율, 기관 순매수

### 2.6 복합 데이터 시계열 관리 (Time-Series Management)
- **대상:** 주가 흐름, 기업 리포트, 재무 정보, 뉴스, 공시 데이터를 단일 시계열 타임라인으로 묶어 통합 관리
- **기대 효과:** 과거 특정 시점의 이벤트-주가 인과관계 분석, 그리고 이를 바탕으로 한 **일일 또는 주간 단위 정기 리포트 자동 생성** 기반 마련

## 3. AI 프레임워크 요건

### 3.1 LangGraph + LangChain 사용 (요건 6)

**사용 이유:**

| 프레임워크 | 역할 | 필요 이유 |
|-----------|------|----------|
| **LangChain** | 데이터 수집·전처리 체인, RAG 파이프라인, 도구 연결 | PDF 로더, 웹 검색, DART API 등 다양한 데이터 소스를 일관된 인터페이스로 연결. 프롬프트 템플릿과 출력 파서로 구조화된 분석 결과 생성 |
| **LangGraph** | 멀티스텝 분석 워크플로우 상태 관리 | 단순 체인으로는 표현 불가능한 "분석 → 질문 생성 → 검색 → 재분석 → 보고서 작성" 사이클형 워크플로우를 상태 그래프로 관리. 각 노드의 성공/실패에 따른 조건 분기 및 루프 처리 |

**멀티 에이전트 (Multi-Agent) 구조:**
데이터 수집, 재무/공시 분석, 프롬프트 엔지니어링, 웹 검색 등 역할을 전문화한 에이전트들이 협력하여 분석의 깊이와 정합성을 높입니다.

**워크플로우 노드 구성 (LangGraph):**
```
[데이터 수집] → [문서 분석] → [질문 생성] → [웹 검색] → [결과 통합] → [보고서 초안] → [검토/반복] → [최종 출력]
```

### 3.2 로컬 LLM - Qwen (요건 7)

| 항목 | 내용 |
|------|------|
| 모델 | Qwen (로컬 실행) |
| 실행 방식 | Ollama 또는 vLLM 기반 로컬 서버 |
| 사용 목적 | 비용 절감, 기업 재무 데이터 외부 유출 방지, 오프라인 환경에서도 동작 |
| 역할 분리 | 1차 분석·분류·요약은 로컬 Qwen 처리, 고도 추론이 필요한 최종 보고서는 필요 시 외부 LLM과 선택적 연동 가능 |

### 3.3 RAG (Retrieval-Augmented Generation) (요건 8)

**사용 이유:**

| 문제 | RAG로 해결하는 방법 |
|------|-------------------|
| LLM의 지식 컷오프 | 최신 애널리스트 리포트·공시·뉴스를 벡터 DB에 색인하여 실시간 컨텍스트 제공 |
| 긴 PDF 문서 처리 | 문서를 청크로 분할·임베딩하여 관련 구절만 선택적으로 LLM에 주입 (컨텍스트 윈도우 절약) |
| 근거 있는 분석 | 생성된 보고서의 각 주장에 출처 문서 및 페이지를 추적 가능하게 함 (환각 방지) |
| 다종 문서 통합 | 리포트·공시·뉴스·재무 데이터를 단일 검색 인터페이스로 통합 조회 |

**벡터 DB:** ChromaDB 또는 FAISS (로컬)

## 4. 핵심 분석 워크플로우 (요건 9)

```
1. 데이터 수집
   └─ 네이버 리포트 PDF + DART 공시 + 재무 지표 + 뉴스

2. 문서 심층 분석 (다회 반복)
   └─ 개별 회사 리포트를 RAG로 여러 차례 조회
   └─ 투자의견 변화·목표주가 트렌드·리스크 요인 추출

3. 자율 질문 생성 (Self-Questioning)
   └─ LangGraph 노드: 분석 공백 탐지 → 후속 질문 자동 생성
   └─ 예: "최근 3개월 외국인 순매도 원인은?", "경쟁사 대비 PER 프리미엄 이유는?"

4. 웹 검색 및 가설 정교화 (Self-Correction)
   └─ DART 공시, 리포트 내용, 주가의 큰 움직임 등을 시스템이 감지하여 스스로 질문을 생성
   └─ 생성된 질문으로 웹 검색을 진행하고, 결과 정보를 교차 검증하여 보고서 내용을 정교하게 보충

5. Human-in-the-Loop (HITL) 개입
   └─ AI 워크플로우 중간 혹은 최종 발행 전, 화면(UI) 시스템 상에서 사용자가 개입하여 내용을 검수하고 피드백/수정을 넣는 대기(Interrupt) 단계 포함

6. 보고서 초안 작성 및 다양한 포맷 지원
   └─ 단일 기업 분석을 비롯해, 프롬프트 파라미터화(동적 템플릿 처리)를 통해 **여러 가지 포맷의 문서**로 변경 출력 가능
   └─ **회사 간 비교 분석 (Peer Comparison)** 등 확장된 리포트 형태 지원
   └─ LangGraph 조건 분기: 품질 기준 미달 시 지시된 루프 재순환
```

## 5. 최종 출력물 (요건 10)

### 5.1 분석 리포트 (Markdown / PDF)

| 섹션 | 내용 |
|------|------|
| Executive Summary | 투자의견, 목표주가, 핵심 근거 3줄 요약 |
| 기업 개요 | 사업모델, 주요 제품/서비스, 경쟁 포지션 |
| 재무 분석 | 핵심 지표 테이블, 최근 3개년 트렌드 |
| 애널리스트 컨센서스 | 최근 리포트 종합, 의견 분포, 목표주가 범위 |
| 리스크 요인 | 단기·중장기 리스크 |
| 결론 및 투자 포인트 | 매수/보유/매도 근거, 모니터링 포인트 |
| 출처 | 참조 리포트·공시·뉴스 링크 목록 |

### 5.2 PPT (PowerPoint / Google Slides)

| 슬라이드 | 내용 |
|---------|------|
| 1 | 표지 (종목명, 날짜, 투자의견) |
| 2 | Executive Summary |
| 3–4 | 기업 개요 및 사업 구조 |
| 5–6 | 재무 핵심 지표 차트 |
| 7 | 애널리스트 컨센서스 요약 |
| 8 | 리스크 요인 |
| 9 | 결론 및 투자 포인트 |

## 6. 시스템 아키텍처 요약

```
[데이터 소스]                [처리 레이어]              [출력]
 네이버 리포트 PDF  ──┐
 DART 공시 API     ──┼──→ LangChain 파이프라인 ──→ ChromaDB (RAG)
 네이버 재무 데이터 ──┤                                   │
 뉴스             ──┘                                    ↓
                                              LangGraph 워크플로우
                                                (로컬 Qwen LLM)
                                                         │
                                                         ↓
                                               Report (MD/PDF) + PPT
```
